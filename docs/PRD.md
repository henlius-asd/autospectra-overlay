## 1. 项目基本信息

* **产品名称**：纯前端离线色谱数据可视化分析工作台
* **版本状态**：正式发布 (V1.0)
* **设计原则**：纯前端离线运行，降低服务器依赖，避免服务端对数据的控制，数据 100% 不离机。

## 2. 项目边界控制

* **核心目标 (Objectives)**：
  1. 100% 纯前端单机运行，零服务器依赖，零网络数据外泄。
  2. 实现 10 条以上曲线（单条最大 100,000 点）的 Waterfall 图叠峦可视化与双向（X/Y 轴）独立微调。
  3. 基于透明 SVG 顶层画布技术，实现横向范围大括号的**动态随动标注**。
  4. 集成 ROI 最大峰对齐及互相关波形对齐引擎。
  5. 支持撤销/重做。
  6. 支持工作区导出/导入（.json）、图表截图导出（.png）、曲线数据导出（.csv）。
* **非目标 (Non-Objectives)**：

  **永久边界（产品定位决定）**：
  1. 不做用户登录鉴权与云端数据备份。
  2. 不直接解析仪器厂家加密的私有二进制文件（如 `.raw`, `.D`），仅接收导出的明文文本（`.arw`, `.txt`, `.csv`）。
  3. 不做仪器直连实时数据采集，本工作台为离线分析工具，数据需提前导出为文本文件。
  4. 不做移动端/平板适配，仅支持桌面端浏览器。
  5. 不做多用户协作与多会话管理。

  **V1.0 暂不做（延后至后续版本）**：
  6. 暂不做纵向分组大括号和 PDF 报告生成。
  7. 暂不做复杂的化学计量计算（如峰面积积分、理论塔板数、自动基线扣除、自动寻峰检测）。
  8. 暂不做自动化批处理/脚本化操作，V1.0 为纯交互式操作。

## 3. 术语表

| 术语 | 定义 |
|------|------|
| **Waterfall 图** | 将多条色谱曲线沿 Y 轴方向依次偏移堆叠，形成类似瀑布的 3D 可视化效果 |
| **ROI 最大峰对齐** | 在指定时间窗口内，以最高峰峰顶为锚点，将其他曲线平移对齐至基准线 |
| **互相关波形对齐** | 通过计算两条曲线在 ROI 窗口内的互相关函数，取最大相关系数处的偏移量作为对齐参数 |
| **大括号标注** | 在色谱图 X 轴方向绘制的大括号图形，用于标注保留时间范围 |
| **偏置 (Offset)** | 人为施加在曲线上的 X/Y 平移量，用于可视化分离重叠曲线，不改变原始数据 |

## 4. 用户角色

| 角色 | 职责 | 核心使用场景 |
|------|------|-------------|
| **QC 实验员** | 日常样品检测，比对色谱图谱 | 导入数据 → 目视检查峰对齐 → 标注保留时间范围 → 截图导出 |
| **方法开发工程师** | 开发色谱分析方法，需要精细调整参数 | 导入多组数据 → 微调偏置 → 切换对齐算法对比效果 → 导出工作区 |

## 5. 页面布局与交互规范

系统采用经典的"左数据、中画布、右工具"三栏式结构，严格桌面端定位（最小分辨率 1366×768，推荐 1920×1080）。

**三栏布局**：
* **左栏（数据区，固定 240px，可折叠）**：
  * 区域拖拽上传（支持批量 `.arw`, `.txt`, `.csv`）。
  * 已挂载文件列表，每条曲线一行：显示/隐藏复选框（眼睛图标）、曲线别名（双击编辑）、颜色指示器。
  * 右键菜单：设为对齐基准线（★ 标识当前基准线）、导出曲线数据为 CSV、删除。
  * 曲线命名规则：默认取文件名（去扩展名）；多列文件取 `文件名_列名`；双击别名可自定义。
  * 列表顶部搜索/筛选框（按名称过滤）。
* **中栏（渲染区，弹性填充）**：
  * ECharts 画布占满底层，支持鼠标滚轮无级缩放（`dataZoom`）、拖拽平移。
  * 顶层叠加透明 SVG Canvas，用于渲染横向大括号标注。
  * **大括号创建**：按住 Shift + 鼠标拖拽框选 X 轴范围 → 弹出气泡输入标签文字 → 确认后大括号渲染在 SVG 覆盖层上，随 `dataZoom` 事件物理粘随。
  * **大括号编辑**：点击已有大括号标签 → 弹出气泡（修改文字 / 删除）；拖拽端点调整范围。
  * 顶部工具栏：撤销/重做按钮、导出截图、导出/导入工作区。
* **右栏（工具箱，固定 320px，可折叠）**：
  * 全局 Y 轴阶梯偏置滑动条（统一拉开曲线纵向间距）。
  * 单线 X/Y 轴高精度独立微调输入框（支持鼠标拖拽数值 + 键盘直接输入，步进 0.001）。
  * 自动对齐模块：
    * 算法下拉切换：ROI 最大峰对齐（适用于主峰明确、无干扰峰的场景） / 互相关波形对齐（适用于基线漂移、峰顶有噪声毛刺、主峰变宽的平台状峰）。
    * ROI 时间窗口：起始时间 / 结束时间输入框 + "从图表选取"按钮（点击后进入选取模式，拖拽框选 X 范围自动填入输入框，选取完成后自动退出选取模式恢复 dataZoom 行为）。
    * 一键对齐执行按钮 + 对齐进度指示器。

**左右栏折叠**：左右栏各有一个折叠/展开按钮，折叠后面板收为窄条（仅显示图标），中栏画布全宽填充，适合全屏查看谱图。

## 6. 关键功能需求详情 (Functional Requirements)

* **【需求 6.1】真实值还原 Tooltip**：传递给图表的数据虽已被加上偏置，但提示框必须在渲染时利用偏置逆向求出 $X_{\text{original}} = X_{\text{rendered}} - X_{\text{offset}}$ 和 $Y_{\text{original}} = Y_{\text{rendered}} - Y_{\text{offset}}$，向 QC 人员呈现真实的仪器响应值。
* **【需求 6.2】SVG 大括号自适应机制**：利用 ECharts 的 `chart.convertToPixel({xAxisIndex: 0}, xVal)`，实时在 `dataZoom` 触发时刷新 SVG 路径数据 `<path d="..." />`，防止图表放大时标注脱节。大括号数据存储结构：`{ id: string; type: 'horizontal'; startX: number; endX: number; label: string }`。
* **【需求 6.3】IndexedDB 持久化与现场还原**：使用 `localForage` 驱动，任何文件的导入和右侧参数的变更必须异步自动序列化至本地数据库。在页面刷新或浏览器关闭重开时，100% 还原先前的全部配置、曲线颜色、大括号标注及对齐状态。原始曲线数据单独存储（IndexedDB 大容量），参数快照（偏置值、大括号、基准线 ID）合并存储。
* **【需求 6.4】文件格式自动检测**：读取文件前 5 行，按以下优先级自动检测：
  1. **分隔符检测**：统计 Tab 和逗号出现次数，取多者为分隔符。
  2. **列数检测**：按分隔符 split 首行，判断列数（2 列 = 单曲线，≥3 列 = 多曲线）。
  3. **表头检测**：若首行包含非数字字符（排除 `#`、科学计数法 `e/E`、正负号），判定为表头行。
  4. **文件头跳过**：自动跳过以 `#`、`//`、`[`、字母开头的注释/元数据行，直到遇到纯数字数据行。
  5. **元数据标签解析**：若文件前若干行为纯字符串标签行（每行一个标签值，均为字符串类型，如样品名、采集日期、波长等），自动按行解析为文件级元数据标签列表，以首个纯数字数据行为标签区与数据区的分界。解析出的标签值在 UI 中展示。
  支持的数据格式：两列制表符分隔 `.txt`、两列逗号分隔 `.csv`、多列 `.csv`（第一列为 Time，后续每列为独立曲线，列名取表头行）、带字符串标签头的 `.arw`。所有格式的**数据行均为横坐标（时间）与纵坐标（具体指标）的数值对**，行内两个数据点间以 TAB 分隔，行与行之间以 `\r` 分隔。解析失败时给出明确错误提示（含行号和错误原因），不静默丢弃数据。解析输出统一为 `ParsedFile { id, name, tags?: string[], curves: CurveData[] }`，其中 `CurveData { name, data: [number, number][] }`。
* **【需求 6.5】对齐基准线管理**：默认第一条成功解析的曲线为基准线（左栏 ★ 标识）。用户可通过右键菜单或点击星标切换基准线。对齐操作始终以当前基准线为参照，其余曲线向其靠拢。基准线 ID 持久化。
* **【需求 6.6】撤销/重做**：基于 Zustand `zundo` 中间件，自动记录 Store 中偏置参数、曲线列表、大括号状态的变化快照。支持 Ctrl+Z / Ctrl+Y。历史栈深度 50 步。原始曲线数据本身不纳入历史（仅存储引用 ID），避免 IndexedDB 膨胀。
* **【需求 6.7】数据导出**：
  * 导出图表截图为 PNG（调用 `echarts.getDataURL()`）。
  * 导出单条曲线数据为 CSV（含原始时间、原始强度、当前偏置后的渲染时间、渲染强度四列）。
  * 导出/导入工作区为 `.json` 文件（含所有曲线数据 + 偏置参数 + 大括号 + 对齐状态 + 基准线 ID），支持跨设备迁移。
* **【需求 6.8】Web Worker 对齐计算**：互相关对齐算法在 Web Worker 中异步执行。主线程发送 `{ refData, targetData, roiStart, roiEnd }` → Worker 返回 `{ xOffset, correlationScore }`。对齐进行中显示非阻塞进度指示器，用户可继续操作其他功能。

## 7. 非功能性需求 (NFR)

* **性能底线**：
  * 滑块联动重绘延迟 < 16ms（确保 60fps），单条曲线最大支持 100,000 个数据点。
  * 10 条曲线 × 50,000 点，互相关对齐（ROI 窗口 ±120 步搜索）总耗时 < 200ms（含 Web Worker 通信开销）。
  * 初始加载（含 IndexedDB 还原）< 3s（10 条曲线 × 50,000 点）。
* **安全合规**：代码中严格禁止存在任何将曲线数据、文件名或批次号发送至外部网络的 `fetch`/`ajax`/`websocket` 行为。系统在彻底断网的单机环境下必须 100% 正常运行。所有依赖（React、ECharts、Tailwind CSS 等）通过 Vite 打包为纯静态资源，不引用任何外部 CDN。
* **浏览器兼容性**：
  * **官方支持**：Chrome 90+、Edge 90+（Chromium 内核）。
  * **尽力兼容**：Firefox 100+（已知问题记录在文档中，不阻塞发布）。
  * **不主动测试**：Safari（实验室场景极少使用 macOS）。
* **分辨率与布局**：严格桌面端，最小分辨率 1366×768，推荐 1920×1080。布局行为见第 5 节。
* **可维护性**：各图层遵循 [ARCHITECTURE.md](ARCHITECTURE.md) 中定义的设计约束，保持函数式解耦。对齐算法扩展遵循 AlignmentAlgorithm 接口契约，为 V2.0 功能增量保留干净的扩展点。
* **数据完整性**：Store 仅持有原始数据与偏置标量，渲染时动态计算偏置后的坐标，确保单一数据源，杜绝派生数据不一致。具体实现约束见 [ARCHITECTURE.md](ARCHITECTURE.md)。

## 9. 风险与假设

| 风险/假设 | 影响 | 缓解措施 |
|-----------|------|---------|
| IndexedDB 在浏览器无痕模式下不可用 | 数据无法持久化 | 检测 IndexedDB 可用性，不可用时降级为 LocalStorage + 明确提示用户 |
| 单文件数据点超过 100,000 时性能下降 | 滑块延迟 > 16ms，用户体验下降 | 利用 ECharts LTTB 采样，超出时自动降采样并提示 |
| 非 Chromium 内核浏览器兼容性问题 | 实验室偶有 Firefox 用户 | Firefox 已知问题记录在案，推荐使用 Chrome/Edge |
