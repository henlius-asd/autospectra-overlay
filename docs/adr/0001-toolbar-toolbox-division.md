# ADR-0001: 工具栏/工具箱功能划分依据

**日期**: 2026-07-15
**状态**: 已采纳

## Context

工具栏 ~25 个按钮平铺一行，无分组无图标，视觉效果拥挤。工具箱仅 3 个面板（元数据、标签样式、自动对齐），名不副实。需确定哪些功能保留在工具栏、哪些迁移到工具箱。

## Decision

按「用户操作时是否需要同时观看图表」划分：

- **工具栏**：保留需边看图表边操作的交互模式（标注、移动、缩放、框选），以及高频文件操作（撤销/重做、导出、工作区管理）。操作时用户视线需保持在图表上。
- **工具箱**：放置配置型功能（显示开关、归一化、层间距），用户操作时通常不需同时观看图表。这些功能低频、配置后即生效，用户可接受视线移开。

## Considered Alternatives

1. **全部留工具栏 + 下拉分组**：工具栏仍承担过多语义，工具箱仍空，偏离 PRD 中"工具箱"的定位。
2. **极简工具栏（仅撤销/导出）**：交互模式迁入工具箱后，用户每次切换模式需移开视线看右栏，不利于边操作边观察图表。

## Consequences

- 工具箱从 3 面板扩到 6 面板，名副其实
- 工具栏从 ~25 按钮降到 ~10-12 图标按钮
- 新增显示设置、数据处理、层间距 3 个面板
- 需要在工具箱中实现 Accordion 手风琴组织以管理 6 面板
