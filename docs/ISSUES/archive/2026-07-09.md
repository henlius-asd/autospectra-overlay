## 功能缺陷

### 1. 纵坐标范围选择（预览图框选）

**状态**：已完成

**优先级**：-

**变更链接**：[y-axis-zoom 规范](../../openspec/specs/y-axis-zoom/spec.md)（含 [2026-07-10-fix-yzoom-default-range](../../openspec/changes/archive/2026-07-10-fix-yzoom-default-range/)）

**问题提出**：黄萍提出纵坐标设置类似横坐标的横条，通过拉动横条选择范围用于放大图叠图。

**原因分析**：目前考虑做成预览图形式，在预览图中框选可视范围。两边都做滑条逻辑较复杂，计划做两个版本（预览图框选方案 vs 双侧滑条方案）比较操作便利性。

**解决方案**：选定「双侧滑条方案」落地——在 `WaterfallChart` 中通过 ECharts 原生 Y 轴 dataZoom slider（`yZoomSlider`，纵向双滑块）实现 Y 轴可见范围框选与平移，范围持久化于工作区 JSON（`yZoomRange`）。预览图框选方案未实现（仅做滑条方案即满足需求）。

### 2. 标签字体大小调整

**状态**：已完成

**优先级**：-

**变更链接**：[add-label-style-editing](../../openspec/changes/archive/2026-07-13-add-label-style-editing/)

**问题提出**：黄萍提出标签里的字体大小设置需要可调整。

**原因分析**：当前标签字体大小固定（`PointLabelOverlay` fontSize=10、`BraceOverlay` fontSize=11、`exportImage` 10/11px 硬编码），用户无法自定义。

**解决方案**：在工具栏新增独立、完整的"标签样式"编辑面板（字号、字体、字重、文字色、背景色），支持默认样式 + 单标签覆盖，渲染与导出共用，纳入 undo/redo 与持久化。详见变更。

### 3. 导出可编辑图片

**状态**：已完成

**优先级**：-

**变更链接**：[pptx-editable-export](../../openspec/changes/archive/2026-07-13-pptx-editable-export/)

**问题提出**：黄萍提出需要导出可编辑图片。

**原因分析**：相对困难，当前导出为静态图片格式（`exportImage.ts` 经 ECharts `getDataURL` 输出位图/SVG，无图层），无法满足后续编辑需求。

**解决方案**：采用"独立 shape"路线——引入 `pptxgenjs`，将每条曲线/坐标轴/刻度/点标签/区间标签重建为 PPT 中可独立选中编辑的 shape，导出 .pptx。详见变更。

### 4. 图例圆点与线条颜色不一致

**状态**：已完成

**优先级**：-

**变更链接**：[fix-legend-color-consistency](../../openspec/changes/archive/2026-07-13-fix-legend-color-consistency/)

**问题提出**：张嫱提出左栏数据区曲线数据旁的原点对应线条颜色，与图例上的圆点和线条颜色不一致。询问能否统一把图例上的圆圈删掉，只体现线条颜色。

**原因分析**：历史遗留问题。`WaterfallChart` legend 未设 `icon`/`itemStyle`，用 ECharts 默认图例（带圆点）；`exportImage` 直接 `legend:{show:false}` 隐藏。

**解决方案**：图例设 `icon:'line'`（仅线段、去圆点），颜色取曲线色；新增"导出含图例"开关（默认关闭），PNG/PPTX 均遵循。详见变更。

### 6. 曲线手动横向/纵向移动（含锁定功能）

**状态**：已完成

**优先级**：-

**变更链接**：[manual-curve-move-and-lock](../../openspec/changes/archive/2026-07-13-manual-curve-move-and-lock/)

**问题提出**：Jenny（邵珍珍）提出能否横向单个样左右移动（该功能可锁定，锁定后无法横向移动），纵向单个样调整间距。

**原因分析**：当前横向移动依赖自动对齐功能完成（`offsets.xOffset` 仅由对齐算法写入），缺少手动微调能力；`CurveScaleOverlay` 的 shift+drag 仅调 `curveScaleOffset`，非持久 offset；无锁定机制。

**解决方案**：新增"手动移动"模式，单曲线拖拽写 xOffset/yOffset；新增 per-curve `locked` 锁定横向（锁定后横向禁用、纵向仍可调）；对齐尊重锁定曲线；纳入 undo/redo 与持久化。详见变更。

### 7. 间隔滑块坍缩为点且无法调整（个别机器）

**状态**：已完成

**优先级**：-

**变更链接**：[fix-layer-spacing-slider-collapse](../../openspec/changes/archive/2026-07-09-fix-layer-spacing-slider-collapse/)

**问题提出**：肖帅报告在个别机器上，间隔滑块坍缩成一个点而不是竖条，且无法改变值，间隔固定为默认最大值 0.5。

**原因分析**：滑块包裹层（`absolute top-1/2 right-1 -translate-y-1/2`，仅设 `top` 无 `height`）高度为 auto，导致其上的 `<input class="layer-slider h-3/5">` 百分比高度被浏览器忽略，回退到内在高度 14px（仅 thumb），轨道塌缩为 0px。经复现测试确认：Bug 版 input 高度 14px，对照组（父级显式高度）180px。

**解决方案**：将百分比高度 `h-3/5` 从 `<input>` 移到包裹层 div（absolutely positioned，百分比解析到确定高度的定位祖先），input 改用 `flex-1` 填充剩余空间。修复后 input 高度 253.2px，可正常拖动。

## 交互体验

### 5. 曲线与坐标刻度轴需有间隔

**状态**：已完成

**优先级**：-

**变更链接**：[split-axis-toggle-and-curve-gap](../../openspec/changes/archive/2026-07-13-split-axis-toggle-and-curve-gap/)

**问题提出**：张嫱提出曲线与坐标刻度轴需有间隔，不要贴线。于璐之前也提出了相同的问题。

**原因分析**：`WaterfallChart` xAxis 未设 `onZero`，ECharts 默认 `onZero:true` 使 X 轴贴 y=0；`computeYAxisRange` 底部仅 2% padding，底层曲线紧贴轴线。同时用户要求 X/Y 轴显示分开、默认只显示 X 轴（当前 `showAxes` 单布尔同控两轴）。

**解决方案**：将 `showAxes` 拆为 `showXAxis`（默认开）/`showYAxis`（默认关）；xAxis 设 `onZero:false` 使轴落至曲线下方；底部 padding 提升至约 8%。详见变更。
