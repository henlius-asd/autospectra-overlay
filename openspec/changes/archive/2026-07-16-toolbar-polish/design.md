## Context

Phase 1 将工具栏从 ~25 个纯文本按钮重构为图标化分组布局。实施后发现：(1) 下拉菜单被图表渲染区遮挡；(2) 多个图标设计与功能语义不匹配；(3) 分组逻辑未按"操作对象"区分标注与变形工具；(4) 导出/导入工作区归属错误。

## Decisions

### 1. 下拉菜单裁切根因与修复

**根因**：`overflow-x: auto` 使 `overflow-y` 从 `visible` 计算为 `auto`（CSS Overflow Module Level 3 §3），导致绝对定位的下拉菜单在垂直方向被裁切。

**修复**：移除 `overflow-x-auto`，添加 `relative z-50`。窄屏溢出问题由 Phase 3 弹性布局 + overlay 机制处理。

### 2. 图标设计原则

所有图标遵循"操作对象 + 操作方向"模式：

- **标注工具**：操作对象是图表区域 → 图标表达"在图表上放置标记"
  - BraceIcon：花括号 `{ }`（区间标注的标准符号）
  - PointLabelIcon：地图定位标记（点标注的标准符号）
  - BoxSelectIcon：虚线矩形（框选的标准符号）
- **变形工具**：操作对象是曲线 → 图标表达"曲线 + 变形方向"
  - MoveIcon：曲线 + 水平双向箭头 ↔（曲线水平位移）
  - ZoomGlobalIcon：多条水平线 + 垂直双向箭头 ↕（所有曲线 Y 轴缩放）
  - ZoomCurveIcon：单条曲线 + 垂直双向箭头 ↕（单曲线 Y 轴缩放）
- **导出工具**：操作对象是文件 → 图标表达"文件类型 + 导出动作"
  - ExportImageIcon：图框 + 下载箭头
  - ExportPptxIcon：演示屏幕 + 折线图

### 3. 工具栏分组逻辑

按"操作对象"分组（方案 C）：

| 分组 | 工具 | 语义 |
|------|------|------|
| 标注工具 | Brace, PointLabel, BoxSelect | 在图表上添加标注 |
| 变形工具 | Move, ZoomGlobal, ZoomCurve | 改变曲线形态 |

变形工具共享视觉语言：曲线 + 方向箭头。标注工具共享视觉语言：标记/区域符号。

### 4. 下拉菜单归属

按操作语义归属：

- **导出 ▾**：生成文件操作（导出图片、导出 PPTX、含图例 checkbox）
- **工作区 ▾**：状态管理操作（导出工作区、导入工作区、新建工作区）

导出/导入工作区是保存/加载应用状态，不是文件导出，归入工作区管理。

## Risks / Trade-offs

| 风险 | 缓解 |
|------|------|
| 移除 overflow-x-auto 后窄屏工具栏可能溢出 | Phase 3 弹性布局 + overlay 机制已处理；后续可添加 ⋯ 更多菜单 |
| 变形工具图标（曲线+箭头）在 18px 下可能不够清晰 | SVG viewBox 24×24 渲染到 18px 仍可辨识；若不够清晰可增大 stroke-width |
